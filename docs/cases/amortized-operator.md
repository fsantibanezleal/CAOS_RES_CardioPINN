# Amortized operator for instant personalization

Vertical id: `amortized-operator` - category: electrophysiology-inverse - lane: replay (the operator input is a
sparse activation-time vector, not a coordinate).

## Medical, biological and physical context

**Medical.** Building a patient-specific cardiac model (a digital twin) today needs a slow per-patient fit. An
amortized operator lets a clinic personalize the model in near real time from routine sparse measurements,
enabling same-visit, simulation-guided decisions.

**Biological.** Every patient heart differs in conduction velocity, fibre architecture and scar. These
parameters determine how arrhythmias arise and respond to therapy, so a useful model must estimate them for
each individual rather than assume population averages.

**Physical.** Instead of solving the inverse problem separately for every patient, an operator is trained once
on a simulated population to map sparse data directly to a parameter posterior (amortized inference). At
inference it returns the estimate and its uncertainty in a single forward pass.

## The research topic

The per-patient inverse verticals (activation mapping, fiber and conductivity, substrate) each fit a fresh
PINN, which takes minutes. For a clinic that maps many patients, an amortized operator is trained ONCE on a
simulated patient population so that a new patient's sparse activation times map in a single forward pass
(milliseconds) to a posterior over the tissue parameters, with an uncertainty. This is the amortized-inference
/ neural-operator direction for cardiac digital twins: no per-patient refit.

## Method

A population of synthetic patients is generated by the Eikonal forward model with random parameters (base
conduction velocity, substrate conduction velocity, substrate location). A heteroscedastic encoder learns the
map from the sparse activation-time vector to the parameter posterior (a mean and a variance per parameter),
trained by the Gaussian negative-log-likelihood. On held-out patients it returns the parameters and the
substrate location in one pass, and the posterior spread is recalibrated so the reported uncertainty is honest.

## Results (measured bake, seed 42, 700 population / 120 held-out)

| Quantity | value |
|---|---|
| Base conduction velocity, normalized RMSE | ~0.007 |
| Substrate conduction velocity, normalized RMSE | ~0.029 |
| Substrate location error | ~0.12 mm |
| Posterior reliability within 2 sigma (raw) | ~0.87 |
| Posterior reliability within 2 sigma (recalibrated) | ~0.93 |
| Inference | ~1 ms (one forward pass) |
| Speedup vs a per-patient fit (~60 s) | ~60000x |

Trained once, the operator recovers the parameters and the substrate location near-exactly with a calibrated
posterior, in one forward pass, roughly four to five orders of magnitude faster than fitting a PINN per
patient. This is the "instant re-mapping across patients" idea realized as an amortized parameter posterior.

## Scope and honesty

- Synthetic population from the Eikonal forward model; the parameterization is low-dimensional (four
  parameters), which the 32 sensors fully constrain, so the recovery is near-exact (a well-posed synthetic
  case). A real population from patient imaging + a higher-dimensional parameterization is the next step; the
  recovery would be harder and the posterior wider. Not clinically validated.
- The operator here is a compact amortized encoder, not a full DeepONet with a branch and trunk; the
  geometry-independent operator state-of-the-art (DeepONet-ViT, kernel operator learning) is documented as the
  direction. The contribution shown is amortized inference with a calibrated parameter posterior.
